TypeScript SDK

Wallet-signed contract integration

@arkafund/sdk wraps generated Soroban bindings with stable modules for the factory, vault, registry, router, oracle guard and venue registry. State-changing calls are built and signed by the integrator's wallet; the SDK never contains a private key. 

npm install @arkafund/sdk

import {
  ARKAFUND_MAINNET_CONTRACTS,
  ArkafundSdk,
  createMainnetConfig,
} from "@arkafund/sdk";

const sdk = new ArkafundSdk(createMainnetConfig());
const factory = sdk.factory(ARKAFUND_MAINNET_CONTRACTS.arkaFactory);
const venues = sdk.venueRegistry(ARKAFUND_MAINNET_CONTRACTS.venueRegistry);

Factory

Atomic Arka creation, creation fee and default risk-policy reads.

Vault

NAV, fee, manager, whitelist, deposit and redemption calls.

Router

Signed route execution with explicit per-hop minimum output.

Venues

Venue status, AUTO eligibility and authorized emergency disabling.

Registry

Discovery pagination, curation and delisting state.

Oracle guard

Asset policy inspection and governed price-policy changes.

Mainnet

Published and traceable contracts

Arkafund's public mainnet manifest lists the contract IDs and SHA-256 WASM hashes used by the current release. The SDK exports the same contract identifiers through ARKAFUND_MAINNET_CONTRACTS.

Catalog API

Live indexer and NAV data

The catalog service continuously indexes public Arkafund mainnet state and exposes health, aggregate metrics and application data. It is a persistent production service; its public endpoints are suitable for monitoring and integrations.

GET/healthService health and synchronization state GET/v1/dashboard/overviewAggregate NAV and catalog metrics GET/v1/monitoring/statusPublic monitoring state and active alerts GET/v1/arkasIndexed Arkas with individual NAV values
