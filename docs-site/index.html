Public APIs

Read indexed Arka data

These endpoints are public and read-only. They do not require wallet permissions and do not custody funds. State-changing actions remain wallet-signed through Stellar contracts.

GEThttps://catalog.arka.fund/healthIndexer health, sync freshness and alert state. GEThttps://catalog.arka.fund/v1/navCanonical aggregate NAV, USD valuation and oracle state. GEThttps://catalog.arka.fund/v1/pricesIndexed USD prices with verified, stale, paused or unavailable state. GEThttps://catalog.arka.fund/v1/metricsTotal Arkas, indexed Arkas, curated Arkas and assets. GET/v1/arkas?curated=true&limit=20Curated Arka list with manager, NAV and asset metadata. GET/v1/dashboard/overviewDashboard aggregates, activity summary and monitoring state. GET/v1/monitoring/statusPublic monitoring status and latest sync run. OPENAPIInteractive API referenceSchemas, parameters, responses and live request execution.

Fetch NAV data

import { CatalogClient } from "@arkafund/sdk";

const catalog = new CatalogClient();
const [health, nav] = await Promise.all([
  catalog.health(),
  catalog.nav(),
]);

console.log(nav.totalNavUsdEstimate, nav.oracleStatus, health.healthy);

Inspect price availability

const prices = await catalog.prices();

for (const price of prices.items) {
  console.log(price.assetContract, price.priceUsd, price.oracleStatus);
}

Fetch curated Arkas

const page = await catalog.arkas({
  curated: true,
  delisted: false,
  limit: 20,
});

for (const arka of page.items) {
  console.log({
    arkaId: arka.arkaId,
    manager: arka.manager,
    denomination: arka.denominationContract,
    nav: arka.nav,
  });
}

API balances and NAV fields are exact on-chain integer strings. Use the asset decimals returned by the API or the manifest metadata before formatting amounts for users.

TypeScript SDK

Build wallet-signed integrations

The SDK exposes typed Catalog and routing clients, wallet-backed product workflows and lower-level contract modules. Every state-changing call is signed by the connected Stellar wallet or the integrator's signer. 

npm install @arkafund/sdk
import {
  ARKAFUND_MAINNET_CONTRACTS,
  ARKAFUND_MAINNET_ASSETS,
  ArkafundSdk,
  createMainnetConfig,
  walletSdkConfig,
} from "@arkafund/sdk";

const network = createMainnetConfig();
const sdk = new ArkafundSdk(
  walletSdkConfig(network, walletAddress, wallet),
);
const workflow = sdk.workflow();

Product workflow

Create and fund Arkas, plan guarded swaps and manage credit positions with human-readable inputs.

Catalog

Typed mainnet reads for health, NAV, prices, Arkas, assets, managers, activity and monitoring.

Routing

Request AUTO or protocol-specific plans with slippage and stateful risk checks.

Factory

Create Arka transactions and inspect creation-fee settings.

Vault

Read vault state and build deposits, redemptions, rebalances and credit actions.

Registry

List Arkas, read manager curation and delisting state.

Router

Build route execution with explicit minimum output controls.

Venue registry

Read enabled venues and protocol-level controls.

Oracle guard

Inspect asset valuation policy and guarded price surfaces.

Token

Read balances and allowances and submit wallet-signed approvals.

Verify indexed state against an Arka contract

const curated = await workflow.catalog.arkas({
  curated: true,
  delisted: false,
  limit: 1,
});
const indexed = curated.items[0];
const inspection = await workflow.inspectArka(indexed.arkaId);

console.log(inspection.consistent, inspection.indexed, inspection.onChain);

Create an Arka and approve its fee

await workflow.createArkaWithFeeApproval({
  denomination: ARKAFUND_MAINNET_ASSETS.USDC,
  managementFeePercent: "1",
  performanceFeePercent: "15",
  depositFeePercent: "0",
  redemptionFeePercent: "0",
  whitelist: [
    ARKAFUND_MAINNET_ASSETS.USDC,
    ARKAFUND_MAINNET_ASSETS.XLM,
  ],
}, approvalExpirationLedger);

Deposit a human-readable amount

await workflow.depositWithApproval({
  arkaId,
  account: walletAddress,
  assetContract: ARKAFUND_MAINNET_ASSETS.USDC,
  amount: "25.50",
}, approvalExpirationLedger);

Plan and sign an AUTO rebalance

const prepared = await workflow.planRebalance({
  protocol: "AUTO",
  amount: "10",
  tokenIn: ARKAFUND_MAINNET_ASSETS.USDC,
  tokenOut: ARKAFUND_MAINNET_ASSETS.XLM,
  slippagePercent: 0.5,
  readerPubKey: walletAddress,
  vaultNav: "1000",
  dailyTurnoverUsed: "0",
  projectedAllocationShiftPercent: "1",
});

await workflow.rebalance(arkaId, walletAddress, prepared);

Manage a Blend position

await workflow.blendLend({
  arkaId,
  adapter: ARKAFUND_MAINNET_CONTRACTS.adapterBlendFixedXlmUsdc,
  marketId: 1,
  assetContract: ARKAFUND_MAINNET_ASSETS.USDC,
  amount: "50",
});

State-changing calls require a wallet-backed signer. Public catalog and contract reads need no API key and never request custody of user funds.
